================================================================================
PROJECT: Inventory Management System (IMS)
================================================================================

OVERVIEW
--------
The Inventory Management System (IMS) is a web-based application designed for
pharmaceutical and manufacturing environments. It manages the full lifecycle of
raw materials, from receipt and quality control through production batching and
finished product labeling. The system ensures traceability, compliance, and
real-time visibility of all inventory movements.

The system is built with a React + Tailwind CSS frontend, a Java Spring Boot
backend, MySQL database, and Keycloak for identity and access management,
all orchestrated via Docker Compose.


WHAT THE SYSTEM DOES
--------------------
The IMS covers the end-to-end material management workflow in a regulated
manufacturing context:

1. A material (raw ingredient, API, excipient, packaging, etc.) is first
   registered as master data in the system.

2. When a shipment arrives, a new inventory lot is created. The lot starts in
   "Quarantine" status and a Raw Material label is automatically generated.

3. Quality Control personnel run tests (identity, potency, microbial, etc.)
   against the lot. When all tests pass, the lot is automatically promoted to
   "Accepted". If any test fails, it is marked "Rejected".

4. Accepted lots can be consumed in production. A production batch is planned,
   raw material lots are assigned as components, and actual usage is recorded,
   which automatically deducts quantities from inventory.

5. When a production batch is complete, a Finished Product label is generated.
   As lots are consumed to zero, they are automatically marked "Depleted".

6. All inventory changes (receipts, usage, splits, transfers, adjustments,
   disposals) are recorded as immutable transactions, providing a full audit
   trail for every lot.

7. Managers and auditors can view real-time inventory dashboards, full
   transaction history for any lot, QC results, and compliance reports.


CORE FEATURES
-------------

Material Master Data Management
  - Create, edit, and deactivate material records (APIs, excipients, dietary
    supplements, containers, closures, process chemicals, testing materials).
  - Each material has a unique part number, type classification, storage
    conditions, and reference to a specification document.

Inventory Lot Management
  - Receive new inventory lots linked to a material.
  - Track per-lot: manufacturer name, manufacturer lot number, supplier,
    received date, expiration date, in-use expiration date, storage location,
    quantity, and unit of measure.
  - Lot lifecycle statuses: Quarantine → Accepted / Rejected → Depleted.
  - Support for splitting a lot to create a child sample lot (is_sample flag,
    parent_lot_id linkage).

Inventory Transactions
  - Every quantity change is recorded as a typed transaction:
    Receipt, Usage, Split, Transfer, Adjustment, Disposal.
  - Each transaction captures who performed it, when, and a reference ID
    (e.g., batch number or purchase order).
  - Immutable history; no direct deletion or editing of past transactions.

Quality Control (QC) Testing
  - Create multiple QC test records per lot.
  - Supported test types: Identity, Potency, Microbial, Growth Promotion,
    Physical, Chemical.
  - Each test tracks method, date, result value, acceptance criteria,
    pass/fail status, and both the performer and verifier (four-eyes principle).
  - Automatic lot status update based on aggregate test results.

Production Batch Management
  - Plan production batches linked to a product (which is itself a material).
  - Assign inventory lots as batch components with planned and actual quantities.
  - Confirm actual usage to automatically deduct stock via Usage transactions.
  - Batch lifecycle statuses: Planned → In Progress → Complete / Rejected.
  - System prevents using lots that are not Accepted, are depleted, or are
    expired.

Label Generation & Printing
  - Configurable label templates per label type:
    Raw Material, Sample, Intermediate, Finished Product, API, Status.
  - Templates store content (HTML/JSON), and physical dimensions (width × height).
  - Labels are auto-generated at key workflow steps:
      * Raw Material label — on lot receipt.
      * Sample label — when a sample lot is split from a parent lot.
      * Finished Product label — when a production batch is completed.
      * Status label — when a lot or batch changes to a significant status
        (e.g., Accepted, Rejected).
  - Operators can reprint any label for an existing lot or batch at any time.

Real-Time Inventory Dashboard & Reports
  - View current stock by material, lot, status, storage location, or
    proximity to expiry.
  - Filter for near-expiry lots (within 30 days).
  - Full traceability: trace any lot through all its transactions and the
    batches it was used in.
  - Compliance reports (QC pass rate, rejected lot counts, usage per batch)
    exportable to Excel/PDF.

Audit Trail & Logging
  - All data-modifying operations are logged with timestamps and user identity.
  - Audit fields (created_by, created_date, modified_by, modified_date) on
    all critical entities.
  - Supports regulatory traceability requirements.

Role-Based Access Control (RBAC) via Keycloak
  - Five roles: Admin, InventoryManager, QualityControl, Production, Viewer.
  - Permissions are enforced per role:
      * Admin — full system access, user management, master data, templates.
      * InventoryManager — receive lots, perform transactions, view reports.
      * QualityControl — create and record QC tests, update lot status.
      * Production — plan batches, assign components, confirm usage.
      * Viewer — read-only access to inventory and reports.
  - Operators cannot approve QC results; QC cannot delete transactions, etc.


DATA MODEL SUMMARY
------------------
The database (inventory_management) contains 8 tables:

  Users               — system users with roles and login tracking
  Materials           — master catalog of all material/product types
  LabelTemplates      — configurable print templates per label type
  InventoryLots       — individual inventory lots with status and quantity
  InventoryTransactions — immutable ledger of all stock movements per lot
  ProductionBatches   — production run records with lifecycle status
  BatchComponents     — links lots to batches with planned vs. actual qty
  QCTests             — quality control test records linked to lots


NON-FUNCTIONAL CHARACTERISTICS
-------------------------------
  - API response time target: < 2 seconds for core operations.
  - Supports at least 100 concurrent users and 10,000 transactions per day.
  - Automatic daily database backup.
  - Health check endpoint with monitoring and alerting (DB latency, disk).
  - Containerized deployment via Docker Compose for consistent environments.
